For Production:
  export GOOGLE_CLIENT_ID="your_client_id.apps.googleusercontent.com"
  export GOOGLE_CLIENT_SECRET="your_client_secret"
  export GOOGLE_OAUTH_REDIRECT_URI="https://yourdomain.com/auth/google/callback"

HOW TO GET CREDENTIALS:

1. Go to Google Cloud Console: https://console.cloud.google.com
2. Create new project or select existing
3. Enable Google+ API
4. Go to Credentials → Create Credentials → OAuth 2.0 Client IDs
5. Select "Web application"
6. Add authorized redirect URIs
7. Copy Client ID and Client Secret

================================================================================
                            HOW IT WORKS
================================================================================

REGISTRATION FLOW:
  1. User clicks "Google orqali kiriting"
  2. Frontend calls GET /api/auth/google/init
  3. Gets Google OAuth URL
  4. Redirected to Google login
  5. User authenticates with Google
  6. Redirected to /auth/google/callback?code=...
  7. Frontend calls POST /api/auth/google/callback
  8. Backend validates code and gets user info
  9. Creates new user or links to existing
  10. Sets secure auth cookie
  11. Redirected to dashboard

LOGIN FLOW:
  - Same as registration but account already exists
  - Google info updated if changed

ACCOUNT LINKING:
  - New Google account + existing email = Link to existing user
  - Preserves user history and settings

================================================================================
                          API ENDPOINTS
================================================================================

GET /api/auth/google/init
  Purpose:      Get Google OAuth URL for frontend redirect
  Auth:         None required
  Parameters:   None
  Response:
  {
    "success": true,
    "authUrl": "https://accounts.google.com/o/oauth2/v2/auth?..."
  }

POST /api/auth/google/callback
  Purpose:      Exchange authorization code for user authentication
  Auth:         None required (code is authorization)
  Body:
  {
    "code": "authorization_code_from_google"
  }
  Response:
  {
    "success": true,
    "user": {
      "id": "user_id",
      "email": "user@example.com",
      "name": "User Name",
      "role": "user"
    }
  }

================================================================================
                          TROUBLESHOOTING
================================================================================

"OAuth is not configured"
  → Check all three environment variables are set
  → Verify they are not empty strings
  → Restart application after setting env vars

"Invalid authorization code"
  → Verify redirect URI matches Google Console exactly
  → Check code hasn't expired (expire after 10 minutes)
  → Verify Client ID and Secret are correct

"Redirect URI mismatch"
  → In Google Console: exact match required
  → No trailing slashes
  → Protocol must match (http vs https)
  → Port must match if not standard

"User creation failed"
  → Check database is accessible
  → Verify Prisma migrations were run
  → Check User model has OAuth fields

Build Errors:
  → Ensure all dependencies installed: pnpm install
  → Run prisma generate: npx prisma generate
  → Check Node.js version (18+)
